## 四、详细步骤

### 步骤一：接听电话（0-30秒）

**动作：**
1. 电话响铃3声内接听
2. 使用标准问候语

**标准话术：**
> "您好，XXX运维紧急报障服务，请问有什么可以帮您？"

**注意事项：**
- 保持冷静、语速适中
- 准备好纸笔或记录系统

---

### 步骤二：核实客户信息（30秒-2分钟）

**核实内容：**

| 核实项 | 说明 | 示例 |
|--------|------|------|
| 客户名称 | 公司/部门全称 | "XX科技有限公司" |
| 联系人 | 来电人姓名及职务 | "张三，运维主管" |
| 联系电话 | 回拨号码 | "138xxxx1234" |
| 客户编码（如有） | 客户唯一标识 | "CUST-001234" |

**标准话术：**
> "请问您是哪家公司？方便留下您的姓名和联系电话吗？"

**记录模板：**
```
客户名称：___________
联系人：___________
联系电话：___________
客户编码：___________（如有）
```

---

### 步骤三：核实问题详情（2-5分钟）

**核实内容：**

| 核实项 | 说明 |
|--------|------|
| 问题现象 | 具体发生了什么 |
| 影响范围 | 影响多少用户/系统 |
| 发生时间 | 问题开始时间 |
| 紧急程度 | 是否影响核心业务 |
| 已采取措施 | 客户已尝试的解决方案 |

**标准话术：**
> "请描述一下具体遇到了什么问题？"
> "这个问题影响了哪些系统或用户？"
> "是什么时候开始出现的？"
> "目前是否影响核心业务？"
> "您这边已经尝试过什么解决方案吗？"

**紧急程度判断：**

| 等级 | 定义 | 示例 |
|------|------|------|
| P0-紧急 | 核心业务中断，影响全部用户 | 系统宕机、数据丢失 |
| P1-高 | 重要功能不可用，影响部分用户 | 关键模块报错 |
| P2-中 | 非核心功能异常 | 次要功能报错 |
| P3-低 | 咨询类问题 | 使用咨询 |

**记录模板：**
```

影响范围：___________
发生时间：___________
紧急程度：P0/P1/P2/P3
已采取措施：___________
```

### 步骤四：确认沟通渠道（1分钟）

**核实内容：**

| 核实项 | 说明 |
|--------|------|
| 是否有钉钉群 | 客户内部钉钉答疑群 |
| 是否有企微群 | 客户内部企业微信群 |
| 群名称/群主 | 便于快速入群 |
| 其他沟通方式 | 邮箱、IM等 |

**标准话术：**
> "请问您这边有钉钉或企业微信的答疑群吗？"
> "群名是什么？群主是谁？"
> "方便拉我进群吗？"

**如无群：**
> "好的，我这边创建一个沟通群，稍后邀请您加入。"

**记录模板：**
```

群名称：___________
群主/邀请人：___________
渠道ID：___________
```

### 步骤五：建联并处理（根据问题复杂度）

**动作：**
1. 加入钉钉/企微群
2. 自我介绍并说明来意
3. 根据问题类型分配处理人
4. 开始排查处理

**标准话术（进群后）：**
> "大家好，我是XXX运维团队的XXX，接到报障电话，现在加入群协助处理问题。"

**处理原则：**
- P0紧急：立即响应，优先处理
- P1高：2小时内响应
- P2中：4小时内响应
- P3低：24小时内响应

## 六、注意事项

### 接听注意事项

1. **态度端正**：保持专业、耐心、同理心
2. **记录完整**：所有信息必须书面记录
3. **确认无误**：关键信息需要复述确认
4. **不轻易承诺**：避免给出无法兑现的时间承诺

### 沟通注意事项

1. 使用客户能理解的语言，避免过多技术术语
2. 及时同步处理进度，避免客户等待焦虑
3. 处理完成后告知客户，确认问题是否解决

### 升级机制

| 情况 | 升级对象 |
|------|---------|
| P0紧急问题 | 立即升级至值班主管 |
| 30分钟未解决 | 升级至技术专家 |
| 2小时未解决 | 升级至部门负责人 |

## 七、常见问题处理

### Q1：客户信息核实不清怎么办？

> 尝试通过来电号码、客户编码等辅助信息确认。仍无法确认的，先处理问题，过程中补充核实。

### Q2：没有钉钉/企微群怎么办？

> 引导客户创建群，或使用电话/邮件沟通，确保问题处理不受阻。

### Q3：客户情绪激动怎么办？

> 保持冷静，表示理解，先安抚情绪再处理问题。必要时请主管介入。

### Q4：问题超出处理能力怎么办？

> 记录问题详情，升级至技术专家或主管，并告知客户预计响应时间。
